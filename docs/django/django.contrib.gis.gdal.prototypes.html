This module houses the ctypes function prototypes for OGR DataSource related data structures. OGR_Dr_*, OGR_DS_*, OGR_L_*, OGR_F_*, OGR_Fld_* routines are relevant here.

django.contrib.gis.gdal.prototypes.errcheck.check_string(result, func, cargs, offset=-1, str_result=False)[source]¶

Checks the string output returned from the given function, and frees the string pointer allocated by OGR. The str_result keyword may be used when the result is the string pointer, otherwise the OGR error code is assumed. The offset keyword may be used to extract the string pointer passed in by-reference at the given slice offset in the function arguments.

Generates a ctypes prototype for the given function with the given argument types that returns a string from a GDAL pointer. The const flag indicates whether the allocated pointer should be freed via the GDAL library routine VSIFree – but only applies only when str_result is True.
