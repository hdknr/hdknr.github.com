class django.contrib.comments.views.comments.CommentPostBadRequest(why)[source]¶

Response returned when a comment post is invalid. If DEBUG is on a nice-ish error message will be displayed (for debugging purposes), but in production mode a simple opaque 400 page will be displayed.

django.contrib.comments.views.comments.post_comment(request, *args, **kwargs)[source]¶

Post a comment.

HTTP POST is required. If POST['submit'] == "preview" or if there are errors a preview template, comments/preview.html, will be rendered.

django.contrib.comments.views.utils.next_redirect(data, default, default_view, **get_kwargs)[source]¶

Handle the “where should I go next?” part of comment views.

The next value could be a kwarg to the function (default), or a ?next=... GET arg, or the URL of a given view (default_view). See the view modules for examples.

Returns an HttpResponseRedirect.
